# Збірка Android APK — покрокова інструкція

Це набагато довший процес, ніж усе, що робили досі. Розбито на фази —
не поспішай, роби по одному кроку.

## Фаза 0: Що взагалі відбувається

Твій месенджер — це веб-сторінка (HTML/CSS/JS), яку раніше показував Electron
(на комп'ютері). Для Android робимо так само, але замість Electron
використовуємо **Capacitor** — він "загортає" ту саму веб-сторінку в
справжній Android-застосунок (.apk).

Сервер (`server.js`) як і раніше має десь працювати — телефон підключається
до нього через інтернет/WiFi, так само як зараз комп'ютер підключається до
`localhost:3000`.

## Фаза 1: Встановлення інструментів

### 1.1. Java (JDK)
Android Studio вимагає Java. Завантаж і встанови **JDK 17**:
https://adoptium.net/temurin/releases/?version=17

Обери: Windows, x64 (або x86, якщо в тебе досі 32-бітна система — але зверни
увагу, що сучасний Android Studio **вимагає 64-бітну Windows**, тому якщо твоя
система 32-бітна, зібрати APK на цьому ж комп'ютері не вийде — доведеться
робити на іншому, 64-бітному ПК).

### 1.2. Android Studio
Завантаж з офіційного сайту: https://developer.android.com/studio

Встанови зі стандартними налаштуваннями (Standard install). Це завантажить
Android SDK автоматично (~3-5 ГБ, займе час).

### 1.3. Node.js
Уже встановлений (той самий, що для десктопної версії месенджера).

## Фаза 2: Підготовка проєкту

1. Розпакуй архів `my-messenger-mobile.zip`.
2. Відкрий термінал у цій папці:
```
cd my-messenger-mobile
npm install
```

3. **Обов'язково** відкрий `www/config.js` і встав правильну адресу сервера
   (дивись коментарі всередині файлу — там пояснено, як дізнатись свою
   локальну IP-адресу через `ipconfig`).

4. Додай Android-платформу:
```
npx cap add android
```
Це створить папку `android/` — це вже повноцінний Android-проєкт.

5. Синхронізуй веб-файли з Android-проєктом:
```
npx cap sync android
```
Роби цей крок щоразу, коли міняєш щось у папці `www/`.

## Фаза 3: Дозволи на камеру й мікрофон (для дзвінків)

Відкрий файл `android/app/src/main/AndroidManifest.xml` і додай ці рядки
всередині тега `<manifest>` (до `<application>`):

```xml
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.CAMERA" />
<uses-permission android:name="android.permission.RECORD_AUDIO" />
<uses-permission android:name="android.permission.MODIFY_AUDIO_SETTINGS" />
```

## Фаза 4: Відкриття в Android Studio і збірка

1. Виконай:
```
npx cap open android
```
Це відкриє проєкт в Android Studio (може зайняти кілька хвилин на перше
відкриття — Gradle завантажує залежності).

2. Дочекайся, поки внизу зникне напис "Gradle sync in progress" / "Indexing".

3. Підключи телефон USB-кабелем (увімкни на телефоні "Налагодження USB" —
   Налаштування → Про телефон → 7 разів тапнути "Номер збірки" → з'явиться
   пункт "Для розробників" → увімкнути USB debugging).
   Або просто збери файл без телефона (крок 4 нижче).

4. **Щоб отримати сам файл .apk** (не встановлюючи одразу на телефон):
   Меню зверху → **Build → Build Bundle(s) / APK(s) → Build APK(s)**

5. Дочекайся завершення збірки (внизу з'явиться сповіщення "APK(s)
   generated successfully" із посиланням **locate** — натисни на нього,
   щоб відкрити папку з готовим файлом.

Файл буде тут:
```
android/app/build/outputs/apk/debug/app-debug.apk
```

Цей .apk можна скопіювати на телефон (через USB, Google Диск, Telegram
"Збережені повідомлення" собі) і встановити (дозволь "Встановлення з
невідомих джерел", якщо Android попросить).

## Важливо: сервер має бути досяжний з телефону

- **Вдома через WiFi**: телефон і комп'ютер в одній мережі — використовуй
  локальну IP-адресу комп'ютера (наприклад `http://192.168.1.50:3000`),
  вказану в `www/config.js`. Через мобільний інтернет (не WiFi) це
  **не спрацює** — телефон не бачить домашню мережу ззовні.

- **З будь-якого місця (мобільний інтернет)**: сервер треба розмістити на
  хостингу з публічною адресою (наприклад, VPS-сервер: DigitalOcean,
  Hetzner, чи будь-який український хостинг-провайдер). Це вже окрема
  задача — скажи, якщо хочеш, підкажу, як розгорнути сервер на хостингу.

## Типові проблеми

- **"Gradle sync failed"** — зазвичай через відсутність інтернету під час
  першого відкриття проєкту (Gradle качає залежності з інтернету). Просто
  почекай і спробуй ще раз (File → Sync Project with Gradle Files).
- **Дзвінки не працюють, хоча чат працює** — перевір, що дав дозволи на
  камеру/мікрофон в налаштуваннях застосунку на телефоні (Android спитає
  сам при першому дзвінку).
- **"App not installed"** при встановленні .apk — увімкни в налаштуваннях
  телефону дозвіл на встановлення з невідомих джерел для того застосунку,
  яким відкриваєш файл (Файли, Telegram тощо).
